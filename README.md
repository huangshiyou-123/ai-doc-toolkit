# AI 文档处理工具箱

> 一个基于 DeepSeek API 的 AI 文档处理工具——在浏览器里粘贴文档，一键生成结构化输出。

## 快速开始

1. 双击 `index.html`（浏览器打开）
2. 点击右上角设置你的 DeepSeek API Key
3. 选择处理模式，粘贴文档，点击生成

## 四种处理模式

| 模式 | 用途 | 示例场景 |
|------|------|----------|
| 📝 文档摘要 | 长文 → 一句话概括 + 要点 + 数据 | 快速了解行业报告核心内容 |
| 📋 需求整理 | 客户原话 → 结构化需求文档 | 客户说了十分钟，整理成一页纸 |
| 🎙️ 会议纪要 | 会议记录 → 标准化纪要 | 从录音转写到待办事项清单 |
| ⚡ 智能识别 | AI 自动判断内容类型 | 不确定类型时使用 |

## 导出格式

- 📋 复制到剪贴板
- 📝 Markdown（.md）
- 📄 纯文本（.txt）
- 🖨️ PDF（浏览器打印）

## 技术栈

- 前端：纯 HTML/CSS/JS（零依赖，单文件）
- API：DeepSeek Chat API
- 存储：API Key 仅存在浏览器本地（localStorage）

## 项目文件

```
ai-doc-toolkit/
├── index.html          # 主程序（打开即用）
├── README.md           # 项目说明
├── 需求整理助手.txt     # Prompt 模板 - 需求整理
├── 文档摘要助手.txt     # Prompt 模板 - 文档摘要
└── 演示材料.pdf        # 需求整理效果演示
```

## 为什么做这个

我是计算机网络专业的学生，对 AI 工具在实际工作场景中的应用有浓厚兴趣。课余自己搭建过 AI 开发环境（DeepSeek + Claude Code），也积累了不少 Prompt 调试经验。这个项目就是把 AI 辅助文档处理的能力做成一个实际可用的工具——面试时能直接在浏览器里跑给别人看。

项目助理、产品助理日常的工作中大量时间在"整理信息"——这个工具要解决的正是这个问题。
