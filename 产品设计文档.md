# AI 文档处理工具箱 — 产品设计文档

> 作者：黄世游 | 日期：2026.06 | 版本：v1.2

## 一、产品概述

**一句话定位：** 把杂乱的文档变成结构化输出的 AI 办公效率工具。

**解决的问题：** 日常办公中大量时间花在"整理信息"上——客户需求要整理、长文档要提炼、会议记录要归纳。每次手动操作重复、耗时，且 AI 输出格式不稳定。

**目标用户：** 项目助理、产品助理、行政人员等需要频繁处理文档的办公人群。

## 二、用户调研

### 2.1 核心用户画像

| 维度 | 描述 |
|------|------|
| 身份 | 项目助理 / 产品助理 / 行政人员 |
| 日常场景 | 每天需要处理 3-5 份不同类型的文档 |
| 核心痛点 | 信息整理耗时；AI 输出格式不稳定；每次要手动写提示词 |
| 期望 | 选一个模式就能用，输出格式固定，能一键导出 |

### 2.2 用户痛点分析

| 痛点 | 严重程度 | 解决方案 |
|------|---------|----------|
| 每次处理文档都要重新写提示词 | 高 | 预设固定模式的 Prompt 模板，选模式即用 |
| AI 输出格式不稳定、排版混乱 | 高 | 给每种模式设计固定输出框架（标题、分点、格式） |
| 输出结果不方便分享和存档 | 中 | 支持复制、Markdown、TXT、PDF 四种导出 |
| 不熟悉 AI 工具 | 中 | 浏览器版双击即用，不需要任何配置 |

### 2.3 我自己的使用观察

在日常使用中我注意到：同样一段客户需求，如果给 AI 不同的提示词，输出的质量差异很大。核心问题不是 AI 能力不够，是**缺少固定的输出框架约束 AI 的行为**。这是我的产品要解决的核心问题。

## 四、功能设计

### 4.1 处理模式设计

每种模式定义了三样东西：**输入规范、输出框架、约束规则**。以需求整理模式为例：

```
【输入规范】
用户输入：客户的原始需求描述（文字，不限格式）

【输出框架】
## 核心需求 → 一句话概括
## 功能点拆解 → 每条不超过两句话，通常 3-7 条
## 待确认的疑问 → 具体、可回答
## 建议下一步 → 1-2 条推进建议

【约束规则】
- 不猜测客户没说清楚的内容，不确定的放入"疑问"栏
- 每条功能点不超过两句话
- 用中文，专业但口语化
```

> 设计思路：这个框架模拟了项目助理收到客户需求后的标准工作流——先提炼核心需求、再拆解功能点、再识别哪些地方需要跟客户确认、最后给出推进建议。本质上是把人的经验变成了 AI 的指令。

### 4.2 交互流程

```
用户打开工具 → 选择处理模式 → 粘贴文档 → 点击"生成"
    → AI 处理中（状态提示） → 输出结构化结果 → 选择导出方式
```

### 4.3 版本设计

| 版本 | 适用场景 | 技术方案 |
|------|---------|----------|
| 浏览器版 | 日常办公，可视化操作 | 纯 HTML/CSS/JS，双击即用 |
| 命令行版 | 批量处理，自动化集成 | Python，纯标准库，管道输入 |

> 设计思路：浏览器版面向"我需要马上处理一份文档"的场景，命令行版面向"我要批量处理文件或接入自动化工作流"的场景。两个版本共用同一套 Prompt 模板，输出格式一致。

## 五、产品原型

### 5.1 界面结构

```
┌──────────────────────────────────────┐
│  AI 文档工具箱          ⚡ API已就绪 │  ← 顶部导航
├──────────────────────────────────────┤
│  📝文档摘要  📋需求整理  🎙️会议纪要  ⚡智能识别 │  ← 模式选择
│  粘贴长文档，AI自动提炼...            │  ← 模式说明
├───────────────────┬──────────────────┤
│  INPUT            │  OUTPUT          │
│                   │                  │
│  [文本输入区]      │  [结构化输出区]   │
│                   │                  │
│                   │                  │
├───────────────────┼──────────────────┤
│ [导入][示例][清空][生成] │ [复制][.md][.txt][PDF] │
└───────────────────┴──────────────────┘
        已就绪 — 选择模式，粘贴文档，点击生成
```

### 5.2 设计原则

- **零学习成本：** 不需要注册、不需要安装、不需要看教程，打开就能用
- **即时反馈：** 生成过程中显示处理状态，生成完成显示耗时和字数
- **隐私优先：** API Key 存储在本地浏览器，不上传任何服务器
- **多模式入口：** 四种模式在顶部一目了然，切换模式时描述文案随之变化
