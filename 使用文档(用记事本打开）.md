> 版本：v2.0 | 更新日期：2026-07-08

---

## 一、快速开始

### 浏览器打开（三种方式任选）

| 方式 | 操作 |
|------|------|
| **在线版** | 浏览器打开 `huangshiyou-123.github.io/ai-doc-toolkit` |
| **本地版** | 双击 `index.html` |
| **exe版** | 双击 `AI文档处理工具.exe` |

打开后无需配置任何东西——已内置免费 API Key，直接就能用。

## 二、四种处理模式

点击页面顶部按钮切换模式：

### 📋 文档摘要

**用于：** 长文章、报告、论文、新闻

**输入：** 粘贴整篇文章

**输出结构：**

| 板块 | 说明 |
|------|------|
| 一句话概括 | ≤30字，主题+结论 |
| 核心要点 | 3-5个独立观点，按逻辑递进 |
| 关键数据 | 表格形式，含数值和含义 |
| 一句话总结 | 实际价值或行动指引 |

---

### 📐 需求整理

**用于：** 客户需求、老板口头要求、聊天记录中的功能描述

**输入：** 粘贴混乱的需求描述

**输出结构：**

| 板块 | 说明 |
|------|------|
| 一句话需求 | 要做[什么]，解决[什么]问题 |
| 背景与目标 | 为什么做 + 做完要达到的效果 |
| 功能点拆解 | 每个功能含优先级（P0/P1/P2）和依赖 |
| 边界与约束 | 明确做的/不做的，技术约束 |
| 待确认问题 | 尖锐提问，直指模糊地带 |
| 建议推进步骤 | 具体的下一步行动 |

---

### 🎙️ 会议纪要

**用于：** 会议录音转文字、会议记录

**输入：** 粘贴会议内容

**输出结构：**

| 板块 | 说明 |
|------|------|
| 会议基本信息 | 表格：主题/日期/参会人/主持人 |
| 议题与讨论 | 每个议题含背景、各方观点、结论 |
| 已决策事项 | 决策内容+决策人+依据 |
| 待办清单 | 复选框格式，含负责人+截止日+优先级 |
| 下次会议 | 时间和议题 |

---

### 🤖 智能识别

**用于：** 不确定类型的内容、混合内容

**输入：** 粘贴任何内容

**输出：** AI 自动判断类型，选择最合适的格式。开头会说明识别出的内容类型。

## 三、文件导入导出

### 📁 导入文件

点击输入框下方的 **📁 导入** 按钮，支持格式：

| 格式 | 说明 |
|------|------|
| `.txt` | 纯文本 |
| `.md` | Markdown |
| `.csv` | CSV表格（自动转文本） |
| `.json` | JSON数据（自动转文本） |
| `.xlsx` / `.xls` | Excel表格（自动提取所有工作表内容） |

### 📤 导出结果

生成结果后，底部按钮一键导出：

| 按钮 | 格式 | 打开方式 |
|------|------|---------|
| 📋 复制 | 剪贴板 | 直接粘贴到任何地方 |
| 📄 Word | `.doc` | Word / WPS 打开，宋体排版，标题带墨绿色下划线 |
| 📊 Excel | `.xls` | Excel / WPS 打开，保留表格样式 |
| 🖨️ PDF | 打印 | 浏览器打印 → 另存为PDF |
| 📝 .md | Markdown | 任何Markdown编辑器 |
| 📄 .txt | 纯文本 | 记事本 |

## 五、常见问题

**Q：需要自己申请API Key吗？**
A：不需要。已内置免费Key，打开即用。如果想用自己的Key（更快更稳定），点左下角「⚡ 设置 API Key」填入。

**Q：支持多长的文档？**
A：建议单次输入不超过 5000 字。超过的话可以分段处理。

**Q：为什么有时输出不对？**
A：AI 输出有一定随机性。如果不满意，点「⚡ 生成」再来一次。输入越清晰，输出越精准。

**Q：我的数据安全吗？**
A：所有处理都通过 API 调用完成，数据不会保存在任何服务器上。API Key 只存在你浏览器的本地存储里。

**Q：Excel导入怎么用？**
A：点 📁 导入 → 选 `.xlsx` 文件 → 自动提取所有工作表 → 内容填入文本框 → 选处理模式 → 点生成。
